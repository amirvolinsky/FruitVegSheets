# fruitVeggieComparator — Google Sheets + Apps Script

## Setup Guide

Rebuilt from the [Python project](https://github.com/amirvolinsky/fruitVeggieComparator) to run entirely inside Google Sheets.

---

## Step 1 — Create the Google Sheet

1. Go to [Google Sheets](https://sheets.google.com) and create a new spreadsheet.
2. Name it: **Fruit & Veggie Comparator**

### Create these sheets (tabs):

| Tab Name | Purpose |
|---|---|
| `DealerPrices` | Weekly expected prices from the dealer |
| `Form Responses` | Auto-filled by the Google Form |
| `ProcessedExpenses` | Derived from form responses (Week + Totals) |
| `WeeklyComparison` | Output — generated by the comparison script |

---

## Step 2 — Set Up Sheet Columns

### DealerPrices

| Column A | Column B | Column C |
|---|---|---|
| **Week** | **ProductName** | **ExpectedPricePerUnit** |
| 2026-17 | Tomato | 5.50 |
| 2026-17 | Cucumber | 3.20 |
| 2026-18 | Tomato | 5.80 |

> **Week format**: `YYYY-WW` (ISO week), e.g. `2026-17` = week 17 of 2026.

### Form Responses (auto-populated by Google Form)

| Column A | Column B | Column C | Column D |
|---|---|---|---|
| **Timestamp** | **ProductName** | **Quantity** | **ActualPricePerUnit** |

### ProcessedExpenses (auto-populated by script)

| Column A | Column B | Column C | Column D | Column E |
|---|---|---|---|---|
| **Week** | **ProductName** | **Quantity** | **ActualPricePerUnit** | **ActualTotal** |

### WeeklyComparison (auto-populated by script)

| Column A | Column B | Column C | Column D | Column E | Column F |
|---|---|---|---|---|---|
| **Week** | **ProductName** | **ExpectedPrice** | **ActualPrice** | **Difference** | **Status** |

## Step 3 — Paste the Apps Script Code

1. In your Google Sheet, go to **Extensions → Apps Script**
2. Delete any existing code in `Code.gs`
3. Copy the entire contents of [`Code.gs`](./Code.gs) and paste it in
4. Click **Save** (💾) — name the project "FruitVeggieComparator"
5. **Close** the Apps Script editor
6. **Reload** the spreadsheet — you should see a new menu: **🍎 Produce Comparator**

---

## Step 4 — Create the Google Form

1. Go to [Google Forms](https://forms.google.com) and create a new form.
2. Title: **Produce Purchase Report**
3. Add these questions (use exact names):

| Question Title | Type |
|---|---|
| `ProductName` | Short answer |
| `Quantity` | Short answer (number) |
| `ActualPricePerUnit` | Short answer (number) |

4. **Link the form to the spreadsheet**:
   - In Google Forms, click the **Responses** tab
   - Click the green Sheets icon (**Link to Sheets**)
   - Select **"Select existing spreadsheet"** → pick your comparator spreadsheet
   - The form will create a "Form Responses 1" tab — **rename it to `Form Responses`**

> **Important**: The form automatically adds a `Timestamp` column. The column order should be: Timestamp, ProductName, Quantity, ActualPricePerUnit.

---

## Step 5 — Create the Form Submit Trigger

1. In your Google Sheet, go to **Extensions → Apps Script**
2. Click the **clock icon** (⏰ Triggers) in the left sidebar
3. Click **+ Add Trigger** (bottom right)
4. Configure:
   - **Function**: `processFormSubmission`
   - **Event source**: From spreadsheet
   - **Event type**: On form submit
5. Click **Save**
6. Grant permissions when prompted

Now every form submission will automatically:
- Convert the timestamp to a week string (YYYY-WW)
- Calculate ActualTotal (Quantity × Price)
- Append to the ProcessedExpenses sheet

---

## Step 6 — Run the Comparison

### Option A — Via Menu
1. Go to **🍎 Produce Comparator → Run Weekly Comparison**

### Option B — Reprocess Form Responses
If you bulk-imported data or corrected form responses:
1. Go to **🍎 Produce Comparator → Reprocess All Form Responses**
2. Then run the weekly comparison

---

## Step 7 — Understanding the Output

The `WeeklyComparison` sheet shows:

| Status | Meaning | Color | Python Equivalent |
|---|---|---|---|
| ✅ Match | Actual price equals expected (within 0.000001) | Green `#DCFCE7` | `✅ תואם` |
| ⚠️ Close | Difference is within 5% tolerance | Yellow `#FEF9C3` | Extended from Python logic |
| ❌ Mismatch | Actual price differs significantly | Red `#FEE2E2` | `❌ לא תואם` |
| 🟡 Missing in Price List | No dealer price found for this week+product | Yellow `#FEF9C3` | `🟡 חסר במחירון` |
| 🟡 No Purchase Recorded | Dealer price exists but no purchase was made | Yellow `#FEF9C3` | N/A (new) |

Below the comparison table, a **Weekly Summary** shows totals per week.

## How the Logic Maps to the Python Project

| Python (`app.py`) | Apps Script (`Code.gs`) | Notes |
|---|---|---|
| `_sanitize_column_name()` | `sanitize()` | Normalizes product names for matching |
| `_find_column()` | N/A | Not needed — fixed column names in Sheets |
| `compare_prices()` — merge on `(מק"ט, תאריך)` | `compareWeeklyPrices()` — join on `(Week, ProductName)` | Same left-join pattern using JS objects as hash maps |
| `pd.merge(..., how='left')` | `dealerMap[key]` lookup + `aggMap` for expenses | O(n) hash lookups instead of DataFrame merge |
| `abs(act - expected) < 1e-6` → `✅ תואם` | `absDiff < CONFIG.EXACT_TOLERANCE` → `✅ Match` | Identical tolerance |
| Default → `❌ לא תואם` | `❌ Mismatch` + new `⚠️ Close` (5% threshold) | Extended with configurable tolerance |
| `pd.isna(expected)` → `🟡 חסר במחירון` | `isNaN(expectedPrice)` → `🟡 Missing in Price List` | Same logic |
| `create_downloadable_excel()` — conditional formatting | Row-by-row `setBackground()` | Same colors: `#DCFCE7`, `#FEE2E2`, `#FEF9C3` |
| `header_fmt` bold + `#F1F5F9` + freeze panes | `setFontWeight('bold')` + `setBackground('#F1F5F9')` + `setFrozenRows(1)` | Identical |

## Performance Notes (10,000+ Rows)

The script is optimized for large datasets:

1. **Single-pass reading**: Each sheet is read once with `getDataRange().getValues()` — one API call per sheet.
2. **Hash maps for lookups**: Both `dealerMap` and `aggMap` use JavaScript objects as hash maps, giving O(1) lookups instead of nested loops.
3. **Batch writing**: Results are written in a single `setValues()` call, not row-by-row `appendRow()`.
4. **Aggregation in memory**: Weekly averages are computed in-memory before any write operations.
5. **Minimal API calls**: The script avoids per-cell operations during the comparison phase. Only the final formatting loop touches individual rows (unavoidable for conditional coloring).

## Configuration

At the top of `Code.gs`, you can adjust:

```javascript
var CONFIG = {
  CLOSE_TOLERANCE: 0.05,   // 5% — threshold for "Close" status
  EXACT_TOLERANCE: 1e-6    // threshold for exact "Match" (from Python)
};
```

Change `CLOSE_TOLERANCE` to adjust sensitivity. Set it to `0` to disable the "Close" status entirely (binary Match/Mismatch like the original Python).
